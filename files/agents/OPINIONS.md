# Seb's Opinions

Durable technical and product opinions, referenced by agents (per `AGENTS.md`) when a decision would benefit from knowing how I think, not loaded by default into every session.

Structure borrowed from [Kun Chen's OPINIONS.md pattern](https://blog.kunchenguid.com/p/everyone-should-have-an-opinionsmd) (the direct origin of this file) and the directive tone of [Jessie Frazelle's AGENTS.md](https://github.com/jessfraz/dotfiles/blob/main/.codex/AGENTS.md) - state the position plainly, then why, so an agent can extrapolate to cases not explicitly covered here.
